Design Goals

The Rosetta system is designed for use by an ornamental turner. It is to:

  1. provide the ornamental turner a way for linking activities using synchronisation of stepper motors, and
  2. assist the ornamantal turner with automating onerous tasks

whilst providing this functionality at a reasonable cost.

The design goal was not to enable a push-button system where the ornamental turner can leave all the work to the computer.

Secondary Goals

The secondary goals were around ease-of-use.

System Interface

The LinuxCNC system is configured like many CNC-based machines in the modern machine shop. This interface is commonly used by many ornamental turners, or is considered easy to learn.

Basic functions are menu-driven, and advanced functionality is available by using gCode.

Use of gCode

Experience has shown that most of the more complex ornamental turning operations are easy to implement in gCode (e.g., adding threading). To that end, we are also providing a library of gCode functions which can be modified by the ornamental turner to achieve the desired output.

Use of Off-the-Shelf Parts

The use of standardly available, off-the-shelf parts is to keep the unit cost down.

The wiring standard we are establishing uses off-the-shelf Ethernet cables, and the system unit has standard RJ-45 plugs where the wires can be easly plugged into the system.

On the other end of that Ethernet cable, where it plugs into the stepper motor, we have also publised standard designs for stepper motor enclosures which can be 3D-printed and which provision for an RJ-45 plug.

Use of Colour Coding

The intent is that the stepper motor enclosures will be printed with the respective colour for the axis on which they are to be used. Then a matching Ethernet cable is used to connect that stepper motor to the base unit.

System Hardware

The Rosetta system is based on

  • LinuxCNC (the version of LinuxCNC on which this help system is based is noted in the header of each page),

  • running on the Debian Linux real-time operating system,

  • which is operating on a Raspberry Pi 5.

The underlying compents are:

  • Mesa 7i92TH, which is a low cost, general purpose, field-programmable gate array (FPGA) based programmable I/O card with a 100 BaseT Ethernet host connection, and

  • DM542T digital stepper drives.
    • Supports peak amp draws from 1.0 to 4.2A. The system is set for 6,400 microstep's per revolution. The spindle's gearing is 9:1, so this results in indexing to about 0.5° of resolution.
    • Supports microstep resolution from 400 to 25,000. The base system is set for 6,400.
    • Supports stepper motors sized from NEMA 17 to NEMA 24. We have tested these motors:
      • Stepper Online's NEMA 17 stepper motor with a single shaft (p/n 17HS19). With 59 Ncm (83.55 oz in) of holding torque, it works quite well.
      • Stepper Online's NEMA 17 stepper motor with a dual shaft (p/n 17HD19). The shaft extending from both sides of the motor is very useful if you want to drive a lead screw with one end, and add a handwheel on the other end for manually moving the driven part.

      • Stepper Online's NEMA 23 stepper motor with a single shaft (p/n 23HS30). With 1.9 Nm (269.06 oz in) of holding torque, this motor works well to drive the spindle.
      • Stepper Online's NEMA 23 stepper motor with a dual shaft (p/n 23HD30).

Further details about the Rosetta system are published in the Rosetta Library.
